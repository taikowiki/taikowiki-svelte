# i18n

This project supports multiple languages. All localized texts can be accessed through the `I18N.i18n` object, and it is designed to use **Korean (ko)** as the default if a corresponding key is missing for a specific language.

## lang File

Create language-specific files in the `$lib/module/i18n/lang` folder.

```ts
// $lib/module/i18n/lang/ko.ts
import type { I18N } from "..";

const ko: I18N.LangFile = {
    layout: {
        main: {
            song: 'Song List',
            ...
        }
    },
    '/song': {
        title: 'Song List',
        ...
    }
}

export default ko;
```

## Features of the i18n System

- **Automatic Fallback**: If you request `i18n.ja.layout.main.song` and the key is missing in the Japanese file, it automatically returns the value from the Korean file.
- **Safe Access**: Accessing a non-existent key does not throw an error; it returns an empty string or a proxy object.

## How to Use

### 1. Language Setup (Layout)

Call `useLang` in the top-level layout to initialize the language state.

```svelte
<script lang="ts">
    import { I18N, useLang, setI18N } from "$lib/module/i18n";
    import { page } from "$app/stores";
    import { writable } from "svelte/store";
    import { setContext } from "svelte";

    const lang = useLang(); // Current language store (ko, en, ja, etc.)
    const { i18n } = I18N;

    // i18n for Layout (Svelte 5 $derived recommended)
    let i18nLayout = $derived(i18n[$lang].layout.main);

    // Set i18n store for pages
    const i18nPage = writable(setI18N($lang, $page.url.pathname));
    setContext("i18n", i18nPage);

    // Update i18n on page navigation
    $effect.pre(() => {
        $i18nPage = setI18N($lang, $page.url.pathname);
    });
</script>

<h1>{i18nLayout.song}</h1>
```

### 2. Using in Pages or Components

Use `getI18N` to retrieve translation data for the current page.

```svelte
<script lang="ts">
    import { getI18N } from "$lib/module/i18n";
    const i18n = getI18N(); // Retrieves the context set in the layout above
</script>

<h2>{$i18n.title}</h2>
```

### 3. Dynamic Routing or Specific Paths

If you need translation for a specific path or are using dynamic routing:

```svelte
<script lang="ts">
    import { getI18N, getLang } from "$lib/module/i18n";
    const lang = getLang();
    
    // Retrieve by specifying a specific key (path)
    let i18n = $derived(getI18N('/song', $lang));
</script>

<p>{i18n.title}</p>
```
