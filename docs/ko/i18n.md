# i18n

이 프로젝트는 다국어를 지원합니다. 모든 다국어 텍스트는 `I18N.i18n` 객체를 통해 접근할 수 있으며, 특정 언어에 해당 키가 없는 경우 **한국어(ko)**를 기본값으로 사용하도록 설계되어 있습니다.

## lang 파일

`$lib/module/i18n/lang` 폴더에 각 언어별 파일을 생성합니다.

```ts
// $lib/module/i18n/lang/ko.ts
import type { I18N } from "..";

const ko: I18N.LangFile = {
    layout: {
        main: {
            song: '곡 목록',
            ...
        }
    },
    '/song': {
        title: '곡 목록',
        ...
    }
}

export default ko;
```

## i18n 시스템의 특징

- **자동 폴백**: `i18n.ja.layout.main.song`을 요청했을 때 일본어 파일에 해당 키가 없으면 자동으로 한국어 파일의 값을 반환합니다.
- **안전한 접근**: 존재하지 않는 키에 접근하더라도 에러가 발생하지 않고 빈 문자열 또는 프록시 객체를 반환합니다.

## 사용 방법

### 1. 언어 설정 (Layout)

최상위 레이아웃에서 `useLang`을 호출하여 언어 상태를 초기화합니다.

```svelte
<script lang="ts">
    import { I18N, useLang, setI18N } from "$lib/module/i18n";
    import { page } from "$app/stores";
    import { writable } from "svelte/store";
    import { setContext } from "svelte";

    const lang = useLang(); // 현재 언어 스토어 (ko, en, ja 등)
    const { i18n } = I18N;

    // 레이아웃용 i18n (Svelte 5 $derived 사용 권장)
    let i18nLayout = $derived(i18n[$lang].layout.main);

    // 페이지용 i18n 스토어 설정
    const i18nPage = writable(setI18N($lang, $page.url.pathname));
    setContext("i18n", i18nPage);

    // 페이지 이동 시 i18n 업데이트
    $effect.pre(() => {
        $i18nPage = setI18N($lang, $page.url.pathname);
    });
</script>

<h1>{i18nLayout.song}</h1>
```

### 2. 페이지 또는 컴포넌트에서 사용

`getI18N`을 사용하여 현재 페이지에 맞는 번역 데이터를 가져옵니다.

```svelte
<script lang="ts">
    import { getI18N } from "$lib/module/i18n";
    const i18n = getI18N(); // 위 레이아웃에서 설정한 context를 가져옴
</script>

<h2>{$i18n.title}</h2>
```

### 3. 다이나믹 라우팅 또는 특정 경로 사용

특정 경로의 번역이 필요하거나 다이나믹 라우팅을 사용하는 경우:

```svelte
<script lang="ts">
    import { getI18N, getLang } from "$lib/module/i18n";
    const lang = getLang();
    
    // 특정 키(경로)를 지정하여 가져오기
    let i18n = $derived(getI18N('/song', $lang));
</script>

<p>{i18n.title}</p>
```
